# Winning Strategy

The highest-leverage move is not adding features. It is making the existing proof easy for judges to trust.

## Lead With Scenarios

Use this order in the submission and video:

1. Human buyer funds an automation benchmark report in escrow.
2. Evaluator resolves the work and creates a receipt trail.
3. Agent operator uses the CLI / JSON path to inspect the same system.

That sequence is easier to understand than opening with standards names.

## Locked Primary Prize Set

1. Synthesis Open Track
2. Protocol Labs: Agents With Receipts / ERC-8004
3. Virtuals: ERC-8183 Open Build
4. Celo: Best Agent on Celo
5. AgentCash / x402 on a testnet-only basis

## Why This Set Works

### Open Track

The product clearly fits:
- agents that pay
- agents that trust
- agents that cooperate

### ERC-8004

The verifier and hook are not cosmetic.
They change who can settle and what trust state is written back after settlement.

### ERC-8183

DealRail is already the commerce and evaluator layer.
This is native architectural fit, not sponsor paint.

### Celo

Celo gives the story multi-chain realism with stable-token settlement and a visible reject path.

### x402

x402 is now worth mentioning because there is real testnet proof, not just adapter code.

## What Not To Say

Do not pitch this as a fully live open marketplace.

The truthful line is:
- escrow, evaluator settlement, receipts, and operator surfaces are real
- some discovery and sponsor integrations remain demo-backed or proof-incomplete

## Best Final Tone

- precise
- conservative
- scenario-first
- proof-heavy

The project should read like a system that already works, not like a future deck.
